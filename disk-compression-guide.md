# Windows 디스크 압축 가이드

## C:\Users 폴더 압축으로 디스크 공간 절약하기

### 디스크 압축이란?

Windows의 NTFS 압축 기능은 파일을 자동으로 압축하여 저장하고, 접근 시 투명하게 압축 해제하는 기능입니다. 사용자는 별도의 조작 없이 일반 파일처럼 사용할 수 있습니다.

---

## 성능에 미치는 영향

### 장점

- **디스크 공간 절약**: 일반적으로 20~60% 공간 절약 가능
- **느린 HDD에서 성능 향상 가능**: 디스크 I/O가 병목인 경우, 읽는 데이터 양이 줄어들어 오히려 빨라질 수 있음

### 단점

- **CPU 부하 증가**: 파일 접근 시마다 압축/해제 연산 발생
- **SSD에서는 거의 이점 없음**: SSD는 I/O가 빠르므로 CPU 오버헤드만 늘어날 수 있음
- **대용량 파일 작업 시 느려질 수 있음**: 동영상 편집, 대형 DB 파일 등

---

## 컴퓨터 유형별 권장 여부

| 저장 장치 | CPU | 권장 여부 | 이유 |
|-----------|-----|-----------|------|
| HDD | 최신 멀티코어 | 권장 | CPU 여유 있고 HDD I/O가 병목 |
| HDD | 오래된 저사양 | 비권장 | CPU 오버헤드 부담 |
| SSD | 모든 사양 | 비권장 | I/O가 빠르므로 이점 없음 |
| NVMe SSD | 모든 사양 | 비권장 | 오히려 성능 저하 가능 |

## 압축 적용 방법

### 방법 1: 탐색기에서 압축 (GUI)

1. `C:\Users\[사용자명]` 폴더 우클릭
2. **속성** 선택
3. **일반** 탭 → **고급** 버튼 클릭
4. **디스크 공간 절약을 위해 내용을 압축** 체크
5. **확인** → **적용**
6. "이 폴더, 하위 폴더 및 파일에 변경 내용 적용" 선택 후 확인

### 방법 2: 명령 프롬프트 (관리자 권한)

```cmd
compact /c /s /i "C:\Users\사용자명"
```

- `/c`: 압축
- `/s`: 하위 폴더 포함
- `/i`: 오류 무시

---

## 압축 해제 방법

탐색기에서 동일한 방법으로 체크 해제하거나:

```cmd
compact /u /s /i "C:\Users\사용자명"
```

---

## 대안 방법 (더 안전한 공간 확보)

압축 대신 다음 방법을 먼저 시도하는 것을 권장합니다:

1. **디스크 정리 실행**: `cleanmgr` 명령으로 임시 파일, 시스템 파일 정리
2. **휴지통 비우기**
3. **Windows 업데이트 캐시 삭제**: `C:\Windows\SoftwareDistribution\Download`
4. **대용량 파일 찾기**: TreeSize Free 등의 도구 활용
5. **OneDrive/클라우드 활용**: 자주 사용하지 않는 파일을 클라우드로 이동
6. **Compact OS 적용**: Windows 자체를 압축하는 방식 (Windows 10/11)

```cmd
# Compact OS 적용 (관리자 권한 필요)
compact /compactos:always
```

## 결론

- **HDD 사용 중이고 CPU가 충분하다면**: 압축 적용해도 성능에 큰 영향 없음
- **SSD 사용 중이라면**: 압축보다 다른 방법으로 공간 확보 권장
- **C:\Users가 전체의 50%를 차지한다면**: 압축보다 대용량 파일 탐색 및 정리가 더 효과적

> **팁**: 압축을 적용하기 전 중요한 데이터는 백업해 두세요.
